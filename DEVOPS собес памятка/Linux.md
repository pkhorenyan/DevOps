
`top` - выводит информацию о работающих в системе процессах и подробности о них
`iostat` - мониторинг использования дисковых разделов

Утилита `iostat`
Наиболее полезные ключи:
-d — отображать только использование дисков
-c — отобразить только использование CPU
-j — отобразить имя раздела (ID, LABEL, PATH, UUID)
-k — отобразить данные в килобайтах
-m — отобразить данные в мегабайтах
-p — отобразить статистику по указанному блочному устройству
-t — отобразить время выполнения теста
-x — отобразить расширенную статистику

**Кейс**: снижается производительность, тормозят все процессы
**Решение:**
Запускаем `iostat`
Смотрим колонку wait. Если значение 100:
Запускаем утилиту `iotop` и смотрим, в чём причина

`iotop` - аналогична утилите top, нов место информации об использовании процессами ЦПУ и памяти показывает работу процессов с дисками
`vmstat` — выводит статистику процессора, памяти и информацию о процессах


Операционная система при своей работе сохраняет всю информацию о работе устройств в директории /proc.

## Процессы

1. Что такое процесс?

- **Процесс** — это экземпляр исполняющейся программы в памяти.
- **Поток (Thread)** — это «легковесный процесс», единица выполнения внутри процесса. У потоков общая память, у процессов — изолированная.
- **PID (Process ID)** — уникальный номер процесса.
- **PPID (Parent PID)** — ID родительского процесса. Все процессы образуют дерево, корнем которого в 2026 году (как и прежде) является **systemd** (PID 1).

2. Жизненный цикл и состояния (States)

Расшифровка букв из команды `top` или `ps`:

- **R (Running/Runnable)** — выполняется или ждет очереди на процессор.
- **S (Interruptible Sleep)** — спит, ожидая события (например, ввода пользователя), может быть прерван сигналом.
- **D (Uninterruptible Sleep)** — «жесткий» сон, обычно ждет ввода-вывода (диск/сеть). Его нельзя убить даже `kill -9`.
- **Z (Zombie)** — процесс завершен, но родитель еще не прочитал код выхода. Занимает только запись в таблице процессов.
- **T (Stopped)** — остановлен сигналом (например, Ctrl+Z).

3. Сигналы (Inter-Process Communication)

Способы общения с процессами:

- **SIGTERM (15)** — мягкая просьба завершиться (по умолчанию в `kill`). Процесс может закрыть файлы и корректно выйти.
- **SIGKILL (9)** — немедленное убийство ядром. Процесс не может его проигнорировать.
- **SIGHUP (1)** — перезагрузка конфигурации (традиционно для демонов).
- **SIGINT (2)** — прерывание с клавиатуры (Ctrl+C).

4. Приоритеты: Nice и PR

- **PR (Priority)** — реальный приоритет, которым оперирует ядро (0-39).
- **NI (Nice)** — «вежливость» процесса от -20 (самый высокий приоритет) до 19 (самый низкий).
    - _Чем выше nice, тем процесс «вежливее» (уступает ресурсы другим)._

5. Основной инструментарий

- **`ps aux`** — моментальный снимок всех процессов.
- **`top` / `htop` / `btop`** — мониторинг в реальном времени.
- **`pstree`** — дерево процессов.
- **`lsof -p <PID>`** — какие файлы открыты процессом.
- **`strace -p <PID>`** — отслеживание системных вызовов (Must-have для отладки).
- **`kill` / `pkill` / `killall`** — отправка сигналов.

6. Файловая система /proc

В Linux «всё есть файл». Информация о каждом процессе лежит в `/proc/<PID>/`.

- Там можно найти: `cmdline` (команда запуска), `environ` (переменные окружения), `fd/` (дескрипторы файлов).
