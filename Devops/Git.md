Как удалить локальную ветку

Вы не можете удалить ветку, на которой сейчас находитесь. Сначала переключитесь на другую (например, `main`). 

- **Безопасное удаление:**  
    `git branch -d <название_ветки>`  
    _Git не даст удалить ветку, если в ней есть неслитые изменения._
- **Принудительное удаление:**  
    `git branch -D <название_ветки>`  
    _Используйте, если хотите удалить ветку вместе со всеми незавершенными изменениями._

Как удалить ветку в GitLab (удаленно)

Если вам нужно удалить ветку не только у себя, но и на сервере:

 ```shell
 git push origin --delete <название_ветки>
 ```

## Merge

Перейдите в ветку, **в которую** вы хотите влить изменения (обычно это `main`), и выполните команду `merge`:

```bash
# Переходим в целевую ветку
git checkout main

# Вливаем изменения из ветки feature в текущую (main)
git merge feature
```

Если во время слияния что-то пошло совсем не так, можно всё откатить командой:

```bash
git merge --abort
```
## Rebase

Rebase из ветки master в нашу нынешнюю - подтягиваем коммиты

```shell
git rebase master
```

## Интерактивный rebase

Укажите, на сколько коммитов назад вы хотите вернуться. Например, для последних 3-х:  
```shell
git rebase -i HEAD~3
```
Или укажите хеш коммита, **стоящего перед** теми, что нужно объединить:

```shell
git rebase -i <hash_коммита>  
```

**Выберите коммиты для объединения**:  
Откроется текстовый редактор со списком коммитов. Они идут в хронологическом порядке (старые сверху).

- Оставьте первый (самый верхний) как `pick`.
- У всех остальных, которые нужно «влить» в первый, замените слово `pick` на `squash` (или просто `s`).

```
pick f7f3f2a Первый хороший коммит
squash 2dc518d Мелкие правки 1
squash 6940e75 Мелкие правки 2
```
